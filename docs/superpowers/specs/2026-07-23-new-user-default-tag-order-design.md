# 新用户默认标签顺序设计

日期：2026-07-23

## 目标

登录用户尚未保存个性化标签顺序时，让“人工智能”成为“综合”之后的第一个标签。“综合”由前端硬编码，不属于后端标签数据。

## 现状与根因

`user_tag_preference` 只保存用户主动提交的排序。新用户没有偏好记录时，`getUserTagOrder` 查询中的所有 `sort_order` 都是 `NULL`，随后按 `tag.id` 升序返回。由于“人工智能”的标签 ID 是 13，所以会排在多个旧标签之后。

## 方案

只修改 `buildGetUserTagOrderSql` 的默认排序规则：

1. 已保存的用户偏好优先，并继续严格按照 `sort_order` 返回。
2. 没有偏好的标签中，“人工智能”排在最前。
3. 其余没有偏好的标签继续按 `tag.id` 升序，保持现有稳定顺序。

这样，新用户没有任何偏好记录时会默认看到“人工智能”第一；已保存完整自定义顺序的用户不会发生变化。若用户只保存了部分标签，已保存部分仍优先，未保存部分再采用上述默认顺序。

## 范围

- 仅影响已登录用户使用的个性化标签顺序接口。
- 游客使用的全局标签列表保持不变。
- 不修改 `tag` ID，不新增数据库字段，不回填或覆盖任何用户偏好数据。

## 测试与验证

- 先增加失败的 SQL 合约测试，要求查询包含“人工智能优先”的无偏好回退排序。
- 运行标签服务测试，确认新增测试通过且原有个性化排序测试不回归。
- 在本地和远程数据库分别用一个无偏好用户执行最终查询，确认“人工智能”位于第一。
- 在远程数据库用已有完整自定义顺序的 Google 用户复核，确认其人工调整后的顺序保持不变。
